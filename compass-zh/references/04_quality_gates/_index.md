# §4 Quality Gates｜驗收 / 自檢 / 工具強制

> 定義「一塊工作真正完成」的硬性閘門：哪些檢查必過、由誰審、審什麼，讓「完成」有客觀標準而非感覺。

## 本章涵蓋

- [01_dod.md](01_dod.md) — Definition of Done：完成的硬性閘門，Required 必過、違反即未完成的鐵律。
- [02_code_review.md](02_code_review.md) — Code Review 三種模式（self／peer／AI）各管什麼、審什麼，以及 AI 審查的信任邊界。

## 何時載入

- 宣稱一塊工作「做完了」之前，要對齊客觀的完成標準。
- 進行 self / peer / AI code review，或想釐清三者分工時。
- 想把驗收標準變成硬閘門、不靠自律時。
