# §9 Collaboration｜cross-person / cross-AI

> When "the user" is really many roles and one PRD spans many sessions: route rulings to the right decider, estimate honestly, and hand off via files so the next person/agent never re-excavates context.

## This chapter covers

- [01_who_decides.md](01_who_decides.md) — Routes each §5 conflict type to the role with authority to rule (eng lead / PM / security), so "await the user's ruling" doesn't collapse into "whoever's loudest in the room."
- [02_estimation.md](02_estimation.md) — Turn a PRD into a credible p50/p90 effort range by counting the checklist and separating known work from research spikes; estimation is a planning aid, not a contract.
- [03_session_handoff.md](03_session_handoff.md) — Zero-friction handoff across sessions/agents: leave the §3.2 tracking docs cold-readable plus a git snapshot so the next in line knows "where we are, what to touch, what not to touch" within 5 minutes.

## When to load

- A PRD conflict surfaces and you need to know *who* actually has the authority to rule on it.
- You're asked to estimate effort for a PRD and want an honest range instead of a single guessed number.
- Context is filling up, work spans days, or another session/agent is taking over the same PRD.
