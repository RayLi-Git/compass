# §9 Collaboration｜跨人 / 跨 AI

> 一份 PRD 落到真實協作場景：誰有權裁決哪類衝突、怎麼把 PRD 估成可信工時、怎麼跨 session / 跨 agent 零摩擦交接。

## 本章涵蓋

- [01_who_decides.md](01_who_decides.md) — 把衝突類型路由到對的裁決者（產品 / 架構 / 安全 / PRD bug / 上線時機），含 solo 一人戴多帽的紀律。
- [02_estimation.md](02_estimation.md) — 把 PRD 轉成 p50/p90 工時範圍：數 checklist、抓未知、分清已知工作與研究探針、套真實乘數。
- [03_session_handoff.md](03_session_handoff.md) — 跨 session / 跨 AI 交接：靠檔案不靠對話，交出方留五樣、接手方冷讀對齊。

## 何時載入

- 衝突卡在「等使用者裁決」卻不知道該問誰、或 solo 模式下強勢帽子壓過安全帽子時。
- 被要求把一份 PRD 估成工時 / 排程，或想拒絕「給我一個準確數字」的壓力時。
- context 將滿、跨日、切 agent，要把進度交給下一棒接手時。
